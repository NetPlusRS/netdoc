Adding a network
The network is added as active. At the next scanner run (run_scanner.py) the CIDR range will be included in the discovery phase (ping sweep + nmap). Devices discovered in this range will be added to the database and visible in the device list. Historical data is not deleted.
Network pause (stop)
The network becomes inactive — the scanner (run_scanner.py) skips this range in future discovery scans. Devices already discovered and saved in the database remain in the system and are still monitored by the ping worker (availability) and SNMP/vulnerability workers. If you want to completely stop monitoring a segment — use the device deletion option. Historical data (scans, logs) is not deleted. Optional: when stopping you can simultaneously delete all devices from this range — they will be rediscovered when the network becomes active again.
Deleting a network
The network configuration (CIDR, note) is removed from the database. The scanner stops scanning this range. Devices previously discovered in this network remain in the database by default (scan history, vulnerabilities, SNMP data). You can choose the option "also delete devices" — then all devices in this IP range will be deleted and if the network is re-added they will be rediscovered.
Note: The host scanner (run_scanner.py) reads active networks from the database at every run. Network status changes will be visible in the next scan cycle (default every 5 min via Task Scheduler).
