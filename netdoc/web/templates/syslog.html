Device IP addresses may not be visible — the IP column shows the Docker NAT address (172.x.x.x) instead of the real network device IP.
Cause: The rsyslog container runs behind the Docker bridge, which masks the source IP of UDP/TCP packets. The fix (network_mode: host) is deployed and works correctly on Linux. On Windows with Docker Desktop the address is still NATed by the WSL2 layer — this is a platform limitation, not a configuration error.
Devices can still be identified via the hostname field in the log message.
